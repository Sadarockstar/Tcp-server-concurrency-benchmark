# TCP Server Concurrency Benchmark (Java)

## Overview

This project benchmarks three different **TCP server architectures** implemented in Java and compares their performance under heavy load using **Apache JMeter**.

The goal is to understand how different concurrency models affect **throughput, latency, scalability, and error rate** when handling a large number of client requests.

The following server implementations were tested:

* **Single-Threaded Server**
* **Thread-per-Client (Multi-Threaded) Server**
* **Thread Pool Server using ExecutorService**

Each server was stress-tested using **600,000 TCP requests** generated by Apache JMeter.

# Server Architectures

## 1. Single-Threaded Server

The server processes **only one client request at a time**.

```
Client1 → processed
Client2 → waiting
Client3 → waiting
```

Characteristics:

* Simple implementation
* No concurrency
* Poor scalability under high load

---

## 2. Multi-Threaded Server (Thread-per-Client)

Each client connection is handled by a **new thread**.

```
Client1 → Thread1
Client2 → Thread2
Client3 → Thread3
```

Characteristics:

* High concurrency
* Better throughput initially
* Can cause **thread explosion** under heavy load

---

## 3. Thread Pool Server (ExecutorService)

A **fixed number of worker threads** handle client requests.

```
Client1 → Worker1
Client2 → Worker2
Client3 → Worker3
Client4 → waiting in queue
```

Characteristics:

* Controlled concurrency
* Prevents excessive thread creation
* More stable under heavy load

# Performance Comparison

| Server Type   | Throughput   | Error Rate | Stability  |
| ------------- | ------------ | ---------- | ---------- |
| Single Thread | 7197 req/sec | 85.34%     | Poor       |
| Multi Thread  | 6118 req/sec | 69.20%     | Unstable   |
| Thread Pool   | 5492 req/sec | **0.41%**  | **Stable** |

Although the single-thread and multi-thread servers show higher throughput numbers, most requests failed.
The **Thread Pool server achieved the best reliability with minimal errors**, demonstrating the advantage of **controlled concurrency**.

---

# Key Insights

This experiment highlights several important systems design principles:

* **Unlimited concurrency can lead to instability**
* Creating a thread for every client can exhaust system resources
* A **thread pool provides controlled concurrency and better stability**
* High throughput numbers are meaningless if the **error rate is high**
